# モバイル軽量化・iPhoneパフォーマンス改善パッチ

「iPhoneメダル モバイル版パフォーマンス調査報告」の内容をもとに、
スマホで重く感じる原因に対して直接効く修正を入れたパッチです。

## 差し替えるファイル

リポジトリ直下の以下3ファイルを差し替えてください。

- index.html
- style.css
- script.js

assets/images と assets/sounds はそのままで大丈夫です。

## 今回の主な修正

### 1. スマホは標準でライト表示に変更

以前は、CPUコア数やメモリが一定以上あるスマホだと重い演出のまま動く可能性がありました。

今回からは、iPhone / Android のようなタッチ端末は原則ライト表示になります。

重い演出を確認したい場合だけ、URL末尾に以下を付けてください。

?full=1

軽量表示を強制したい場合は以下です。

?lite=1

### 2. スマホで重いCRT演出を削減

スマホでは以下をかなり減らしました。

- 走査線
- CRT反射
- blend系の重ね描画
- 多重シャドウ
- 一部の発光
- 常時アニメーション

PCでは従来寄り、スマホでは操作感優先になります。

### 3. 起動直後の画像プリロードを削減

以前は起動直後に、今すぐ使わないキャラクター差分やシーン画像を裏で読みに行く可能性がありました。

今回からスマホでは、必要になった時に読む方式へ寄せています。

### 4. 画像decodeをスマホでは軽くしました

非表示画像の大量 decode は iPhone で重くなりやすいため、スマホではブラウザ任せにして負荷を下げています。

### 5. 初回タップ時の音声warm-upを軽量化

初回タップ時にSFXをまとめて load する処理をスマホでは避けました。
音の準備でゲーム開始が重くなる問題を減らします。

### 6. ギャラリー画像の裏プリロードをスマホでは停止

ギャラリーを開いていない時に、裏で画像を読みに行く処理をスマホでは止めています。

## 確認方法

通常URLで確認：

https://noapi2042-bit.github.io/janken-retro-game2.5/

軽量表示を強制：

?lite=1

重い演出を強制：

?full=1

キャッシュが残っている場合：

?reset=1

## 重要

このパッチは、まず体感の重さを落とすための即効性重視です。
さらに軽くするには、次の段階で画像自体を WebP / AVIF 化したり、
キャラクター差分を整理するのが効果的です。

## 構文確認

node --check script.js
