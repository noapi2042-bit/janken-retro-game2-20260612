# 画像・BGM・スコア受け取り・チュートリアル導線調整パッチ

今回のパッチは、以下を中心に修正しています。

- スマホ軽量モードでも BGM がチャンス/ファイナルで切り替わるように修正
- キャラクター画像が出ない時の共通フォールバック画像を追加
- あいこで増えたメダルが、勝った側のスコアへそのまま入るように修正
- 1回目の普通じゃんけんの目的表示を強化
- 2回目/3回目のチュートリアル区間を少し長めに調整
- 最終じゃんけんを「ノーヒント＋再確認あり」として分かりやすく調整
- 「ワナ」っぽい表現を減らし、「ためす / 見せた手を外す」に整理

## 差し替えるファイル

リポジトリ直下の以下3ファイルを差し替えてください。

- index.html
- style.css
- script.js

assets/images と assets/sounds は基本そのままで大丈夫です。

## BGM切り替えについて

前回までは、スマホ軽量モードでは重さ対策として通常BGMだけを使う状態になっていました。
そのため、AQUOS wish / iPhone ではチャンスやファイナルに入ってもBGMが切り替わらないことがありました。

今回からは、スマホでも必要になった瞬間だけBGMを読み込み、以下へ切り替えます。

- 通常: bgm_loop.mp3
- チャンス: bgm_chance.mp3
- ファイナル: bgm_final.mp3
- TRUE END: bgm_true_end.mp3

## 画像について

キャラ画像が出ない場合、まずその表情の画像を探し、見つからなければ以下の共通立ち絵を試します。

- character_draw_01.png
- character_normal_01.png
- character_happy_01.png

それでも無い場合だけ、緑のフォールバック顔を表示します。

### 判断

- 手画像 0.03MB 前後: 問題なし
- キャラ立ち絵 0.18MB 前後: 問題なし
- シーン絵 2MB 前後: lazy読み込みなら許容範囲。ただし増えすぎると重くなります
- BGM 3〜7MB: スマホでは重めです。可能なら 64〜96kbps へ圧縮推奨です

## スコア修正

今までは、勝った側のスコアが30で止まって見えることがありました。
今回からは、あいこで増えたメダルを勝った側がそのまま受け取ります。

例：

- メダル 80 で勝つ → あなた +80
- メダル 160 で負ける → あいて +160

勝敗判定は今まで通り、どちらかが30以上になった時点で終了します。

## チュートリアル導線

普通勝利後の練習区間を少し長くしました。

- チャンス: あいこ7回
- ファイナル: 基本あいこ12〜14回

最終じゃんけんはノーヒントにして、1回目タップで確認、同じ手をもう一度押すと決定します。
