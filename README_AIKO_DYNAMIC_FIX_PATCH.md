# ためす判定の再修正パッチ

## 修正した問題

「きもち：ためす」で、

- セリフ：ぱーっぽく見えるかな？
- プレイヤー：ぐー

を選んだ時、本来は「ぱー以外を選べたので、あいてがぐーに合わせてあいこ」になるべきところ、勝ちになる場合がありました。

## 原因

今回の仕様では、ためすは「固定された相手の手を当てる」のではなく、

- 見せ手を避けたら成功
- 成功時は、あいてがプレイヤーの手に合わせる

という特殊判定です。

ただし、ラウンド開始時の表示更新や、以前のキャッシュ/古いイベント情報が混ざると、通常の固定CPU手として処理される可能性が残っていました。

## 対応

### 1. ラウンド開始時点の心理イベントを退避

手を選んだ瞬間の `state.psychEvent` を `activePsychEvent` として確保してから、画面更新を行うようにしました。

### 2. chooseCpuHand を堅牢化

`dynamicMode` が残っていない場合でも、

- `feeling === "bait"` なら `avoid`
- `feeling === "mirror"` なら `mirror`

として扱います。

### 3. ためす成功時は必ずあいこ

ためすでは、

- 見せ手以外を選んだ → `cpuHand = player`
- 見せ手を選んだ → あいてが勝つ手を出す

に固定しました。

## 例

- セリフ：ぱーっぽく見えるかな？
- きもち：ためす

結果：

- ぐー → あいてもぐー → あいこ
- ちょき → あいてもちょき → あいこ
- ぱー → ひっかかった扱い
