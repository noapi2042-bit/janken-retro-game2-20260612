# モバイル監査ベース修正版

前回のモバイル調査報告を元に、現在の `index.html / style.css / script.js` へ再適用した修正版です。

## 差し替えるファイル

- index.html
- style.css
- script.js

`assets/images` と `assets/sounds` はそのままで大丈夫です。

## 主な修正

### 1. スマホは標準で軽量モード

AQUOS wish SH-51C、iPhone、Android、タッチ端末は基本的に軽量モードへ倒します。
`?full=1` を付けると重い演出ありで確認できます。

### 2. キャラ画像未表示対策

画像読み込み失敗時に失敗状態をキャッシュしないようにしました。
また、候補を `.png / .webp / .jpg / .jpeg / .png.png` まで広げています。

### 3. 画像読み込みを軽量化

`decoding="async"`、`loading`、`fetchPriority` を使い、スマホでは必要な画像だけ読む方向へ寄せています。

### 4. 起動時の先読みを削減

軽量モードでは、起動直後に大量のキャラ差分・シーン画像・アルバム画像を先読みしません。

### 5. 音声初期化を軽量化

軽量モードでは初回タップ時のBGM/SFX一括ロードを避け、必要最低限にしています。

### 6. CSS軽量化

軽量モードでは、走査線、反射、重い影、発光、疑似要素、アニメーションを大きく削っています。

## 確認方法

通常確認：そのまま開く

軽量モード強制：

```text
?lite=1
```

重い演出確認：

```text
?full=1
```
